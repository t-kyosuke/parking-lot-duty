## ★当番割り当てアルゴリズム（最重要）

> **2026-05-31 改定**：「**累計回数が少ない人を優先**」する方式に変更（旧：固定順リストをポインタで循環させる方式）。
> 実装は `src/lib/assignParking.ts` の `assignDuties` / `pickByCount`。

### 基本の考え方

各当番枠（駐車場・ビデオ・カゴ）について、その日に出席（◯）しているコーチの中から、**これまでの累計担当回数が最も少ない人**を選ぶ。これにより全員の担当回数が均等化される。

> **2026-06-16 追加 → 2026-06-22 改定（カゴ連鎖）**：カゴ（黄色うちわ＋道具が入った“物理的に1個”の物）を運ぶ役を、**試合日だけでなく「カゴが必要な全セッション（土日祝の練習＋試合。合宿/休みは除外）」**について、**日付順のバトンリレー**として割り当てる。実装は `src/lib/assignParking.ts` の `assignKagoChain`（`assignDuties` で駐車場/ビデオを確定した**後**にその結果を読むだけ＝**駐車場/ビデオの当番者は一切変えない一方通行**）。詳細は下の「補足ルール」のカゴ項を参照。
> 変更前の仕様は `docs/SPEC-SNAPSHOT-before-saturday-kago.md` に記録、復元ポイントは git tag `pre-saturday-kago`（戻したくなったらここへ）。

> **2026-07-01 改定（カゴ表示を「持ち帰り」基準に。※表示だけ・割り当てロジックは不変）**：カゴの割り当て（`assignKagoChain`／`kagoCoach`＝「その日カゴを持ってくる人」）は一切変えず、**画面・LINEの見せ方だけ**を「**その日の練習後に誰がカゴを持ち帰るか**」に変更。物理的にカゴはバトンなので「その日の練習後に持ち帰る人」＝「**次のカゴ利用日にカゴを持ってくる人**」。この導出は純粋関数 `computeKagoTakeHome(assignments)` が担う（次のカゴ利用日＝日付順で次のセッション。無ければ「翌月へ引き継ぎ」、次が要確認なら「要確認」）。**カゴ連鎖導入前（新フラグ無し）の旧・確定月データは新方式が成立しないので、従来どおり「その日のカゴ担当（試合日のみ）」を表示し担当のいない日はカゴ行を出さない、へ自動フォールバックする**（4〜6月など旧データ月を開いても誤表示しない）。
> - **公開画面・LINE**：カゴが必要な全日に **「🧺当日カゴ持ち帰り▶◯◯さん」** を毎回表示（日曜で駐車場当番が持つ日も隠さず出す）。
> - **管理画面（結果表）**：編集対象は従来どおり **「🧺カゴを持ってくる人▶」（＝`kagoCoach`）** のまま。その下に読み取り専用で **「→終わりに◯◯さんが持ち帰り」** を補足表示。
> - **公開画面のラベル**は3行を8文字で統一：**🅿️当日の駐車場当番／🎥当日のビデオ当番／🧺当日カゴ持ち帰り**（CSSはPC＝横並び、スマホ＝ラベル→値の縦積みで横あふれ回避）。
> - 復元ポイントはこの改定直前のコミット（表示だけの変更なので `git revert` で戻せる）。

判定の優先順位（`pickByCount`。上から順に絞り込み、候補が0人になったらその条件だけ緩める）：
1. **出席（◯）している人だけ**が候補（△・×・空欄は対象外）
2. **被り防止**：同じ日にすでに駐車場当番になった人は、ビデオの候補から外す
3. **月内上限**：その月すでに上限回数（既定 `DEFAULT_MONTHLY_LIMIT` = 2回）当番した人は外す（累計0の人が単月で追いつき過ぎる＝連続はしないが頻度が高くなるのを防ぐ）
4. **連続防止**：直前に同じ当番をした人は外す
5. 残った候補のうち**累計回数が最少の人**。同数なら固定順（`COACH_ORDER`）で先に来る人

補足ルール：
- **駐車場**：日曜・祝日のみ（土曜は割り当てなし＝`null`）。練習日（`practice`/`special`）が対象
- **ビデオ**：全日（土曜も含む）。同じ日の駐車場当番とは被らない。練習日が対象
- **カゴ（連鎖／バトンリレー。`assignKagoChain`）**：カゴが必要な全セッション（練習 `practice`/`special` ＋ 試合 `match`。合宿/休みは除外）を**日付順**に処理し、各日「カゴを持ってくる人」を1人決める。カゴは物理的に1個なので**「前回のカゴ・セッションに出席◯だった人」しか次に持って来られない**＝候補は「**今日◯ ∩ 前回◯**」。
  - 日曜・祝日の練習（駐車場あり）で**駐車場当番が前回も◯**なら、その人がそのまま運ぶ（うちわ入りのため）。これは**カゴ係に数えない**（`kagoCarriedByParking=true`）。※表示上は2026-07-01改定で毎日「持ち帰り」を出すため、この日も隠さず表示する（上のカゴ表示の注記を参照）。
  - それ以外（土曜・試合・上記で駐車場当番が前回いなかった日）は**カゴ係を1人指名**（累計に数える）。同日の駐車場・ビデオはなるべく避け、累計最少・連続回避で選ぶ。
  - 「今日◯ ∩ 前回◯」が0人＝**要確認**（`kagoNeedsConfirm=true`）。`kagoHolder` に現在の保持者（連絡先）を出して手動で引き継ぐ。要確認の日は保持者を据え置くので**連鎖は途切れない**。
  - 月またぎは `getPreviousKagoSession`（前月最後のカゴ・セッションの保持者＋その日の出欠）で引き継ぐ。累計は `srs_kago_counts`（**カゴ係指名分のみ**＝`isKagoCounted`。兼任・要確認は数えない）。`AssignmentResult` の `kagoCoach`/`kagoCarriedByParking`/`kagoNeedsConfirm`/`kagoHolder`/`isMatch` で保持。**駐車場・ビデオの公平さは一切いじらない（一方通行）**。
- **累計は月をまたいで引き継ぐ**（`getCountsForAssignment` が「現在累計−その月の確定分」を返す＝再割り当てでも二重カウントしない。駐車場/ビデオ/カゴの3種に対応）
- **連続防止の起点**は前月末の当番者（`getPreviousLastCoach` が選択月より前の確定済み月から取得）
- **月内上限**：同じ人は1か月に既定2回まで（`DEFAULT_MONTHLY_LIMIT`）。月内カウントは月ごとにリセット。`assignDuties` の引数で変更可

```javascript
// 1枠ぶんの当番者を選ぶ（核心）
function pickByCount(order, counts, dayAtt, lastCoach, conflictCoach, monthly, limit) {
  const present = order.filter(c => dayAtt[c] === '◯' && c !== conflictCoach);
  if (present.length === 0) return null;                     // 出席者なし＝該当者なし
  let underLimit = present.filter(c => (monthly[c] ?? 0) < limit); // 月内上限
  if (underLimit.length === 0) underLimit = present;               // 全員上限なら緩める
  let candidates = underLimit.filter(c => c !== lastCoach);        // 連続防止
  if (candidates.length === 0) candidates = underLimit;           // 尽きたら緩める
  let best = candidates[0]; // 累計最少（order順なので厳密に小さい時だけ更新＝同数はorder先頭）
  for (const c of candidates) if ((counts[c] ?? 0) < (counts[best] ?? 0)) best = c;
  return best;
}
```

## CSVパース仕様

### 調整さんCSVの形式
```csv
2025年SRS3年コーチ出欠確認
コーチ陣の出欠確認こちらでお願いします
日程,茂木隼人,橋戸佑介,外山岳,...
3/29(日) 関大ラグビーフェスティバル（仮）,◯,◯,△,...
4/4(土) 13:40-16:40,◯,◯,×,...
...
コメント,,,祝日、日曜日は...,...
```

### パースルール
- **先頭2行**：メタデータ → スキップ
- **3行目**：ヘッダー（コーチ名の列）
- **最終行「コメント」**：スキップ
- **空欄（`,,`）**：△（未定）と同等に扱う
- **名前照合**：スペースを除去して比較（「岸下 和樹」→「岸下和樹」）
- **同日複数行**：最初の行の出欠を使用
- **エンコーディング**：UTF-8（Shift_JISにもフォールバック対応）
- **日程列**から練習時間を自動抽出（例：`4/5(日) 13:00-14:20` → `13:00-14:20`）

### LINEアナウンス出力形式
練習日は「駐車場▶／ビデオ▶／🧺当日カゴ持ち帰り▶」、試合日は「⚽試合：🧺当日カゴ持ち帰り▶」を日付順に出力する。**カゴ行（🧺当日カゴ持ち帰り）は毎日出す**（＝その日の練習後にカゴを持ち帰る人。日曜で駐車場当番が持つ日も出す）。月内最後のカゴ利用日は「翌月へ引き継ぎ」。
```
7月の当番は下記でお願いします。

各日の「🧺当日カゴ持ち帰り」の人が、その練習後に黄色うちわ&カゴセットを持ち帰ります。

7月4日（土曜日）：駐車場▶-　/　ビデオ▶国沢さん　/　🧺当日カゴ持ち帰り▶国沢さん
7月5日（日曜日）：駐車場▶国沢さん　/　ビデオ▶堀本さん　/　🧺当日カゴ持ち帰り▶堀本さん
7月12日（日曜日）：駐車場▶岸下さん　/　ビデオ▶堀本さん　/　🧺当日カゴ持ち帰り▶国沢さん
7月26日（日曜日）：駐車場▶松木さん　/　ビデオ▶大串さん　/　🧺当日カゴ持ち帰り▶翌月へ引き継ぎ

よろしくお願いします。
```
（例：7/4の練習後は「国沢さん」がカゴを持ち帰り、次のカゴ利用日=7/5に持ってくる。最後の7/26は翌月の初回担当が未確定なので「翌月へ引き継ぎ」）
