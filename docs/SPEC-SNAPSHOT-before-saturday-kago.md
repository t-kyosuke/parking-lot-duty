## カゴ当番（変更前）の決め方

- **対象は「試合の日」だけ**。練習日（土曜・日曜・祝日）にはカゴ当番は決めていなかった（`kagoCoach = null`）。
- 試合日のカゴ当番は **その試合に出席（◯）している人の中で、これまでのカゴ累計が最少の人**。同数なら固定順（`KAGO_COACH_ORDER`）の先頭。直前のカゴ当番者は連続を避ける。
- **「試合前の練習に出ていたか（＝実際にカゴを持っているか）」は見ていない**（＝持っていない人を指名し得る穴がある）。
- 実装：`src/lib/assignParking.ts` の `assignKagoDuties`（`matchDays` のみ・当日出欠のみ参照）。

### 運用（人の手でやっていたこと）
- カゴ（黄色うちわ＋道具）は **次に駐車場当番をやる人が前回の練習で持ち帰り**、自分の当番日に持ってくる。
- その人が欠席なら、別の誰かが善意で持ち帰っていた。
- 土曜練習のカゴは「誰が持ってくるか」決めておらず、善意任せだった（← これが今回の変更の動機）。

## 駐車場・ビデオ当番（変更前 ＝ 今回も変えない）

- **駐車場**：日曜・祝日の練習のみ（土曜はなし）。出席◯の中から累計最少・月内上限2回・連続回避。
- **ビデオ**：全練習日（土曜含む）。同上。同じ日の駐車場当番とは別人。
- 実装：`assignDuties` / `pickByCount`。**今回の変更でここは1行も触らない。**

## データの形（変更前）

`AssignmentResult`（`src/lib/assignParking.ts`）:
```
date, dayOfWeek,
coach: string | null,       // 駐車場（土曜・試合日は null）
videoCoach: string | null,  // ビデオ（試合日は null）
kagoCoach: string | null,   // カゴ（試合日のみ。練習日は null）
practiceTime, isSaturday, isMatch
```

累計の保存（localStorage）:
- `srs_parking_counts` / `srs_video_counts` / `srs_kago_counts`
- カゴ累計は **試合日のカゴ当番だけ** を数えていた（`recalculateCumulativeCounts` が `kagoCoach` を無条件に +1）。

## 表示・LINE（変更前）

- **AssignmentResult.tsx / PublicView.tsx**：試合日のみ「🧺カゴ▶◯◯さん」を表示。練習日は駐車場/ビデオの2枠のみ。
- **LineAnnouncement.tsx**：試合日のみ「⚽試合：🧺カゴ▶◯◯さん」。練習日にカゴ行なし。

## 「戻し方」（前の状態に戻す手順）

1. `git`（または開発者）で tag `pre-saturday-kago` の状態に戻す。
2. `npm run build` で本番用にまとめ直す。
3. 管理画面の「🌐 スマホ向けに公開する」で再公開する。
4. 累計（`srs_kago_counts` 等）が新方式で変わっている場合は、設定タブの「累計回数の調整」で必要に応じて手直しする。

> 過去に確定・公開済みの月（LINEで流した分）は、新方式でも **数値の意味を保ったまま** 引き継ぐ設計にしているため、原則そのまま。
